# PRD：AI Agent 任务执行可视化面板（Web 端）v1.0

> 演示用 PRD，可直接整段粘贴到 NEXUS 聊天框作为输入。
> 业务参考：Cursor Background Agents / Devin / Manus / Claude Code Web。

## 1. 背景

随着 Background Agent 模式（Cursor / Devin / Manus）兴起，研发开始把 LLM Agent 当作"异步执行的初级工程师"使用：发出任务后离开，定期回来 Review 进度。当前 NEXUS 平台缺少一个 Web 端的多任务执行可视化面板，导致用户必须依赖终端日志或后端接口排查。

## 2. 目标

为 NEXUS 平台用户提供一个 Web 端的 Agent 任务执行控制台，支持同时观察、控制、回放多个并发的 Agent 任务，目标用户为开发者和 PM。

## 3. 核心用户故事

- 作为研发，我希望同时跑 5 个 Agent 任务（修 bug、重构、写测试、写文档、查日志），并在一屏上观察所有进度。
- 作为研发，我希望能暂停某个跑偏的任务并在中途插入新的指令。
- 作为 Reviewer，我希望能回放任务的整个执行过程，看到 Agent 调用了哪些工具、修改了哪些文件、写了哪些代码。
- 作为 PM,我希望能把跑完的任务结果分享出去（链接形式）。

## 4. 功能需求

### 4.1 任务列表（左侧面板，宽度 360px）

- 卡片瀑布流，按更新时间倒序
- 每张卡显示：任务标题、Agent 名、当前状态、进度条、最近一条 log（一行）
- 状态有：queued / running / paused / completed / failed
- 支持筛选（状态 / Agent 类型 / 时间范围）和搜索（按标题模糊）
- 任务数 > 50 时启用虚拟滚动
- 列表上方有"新建任务"按钮，点击弹出抽屉

### 4.2 任务详情（右侧主区，自适应宽度）

- 头部：标题、Agent、状态、计时器、操作按钮（暂停 / 继续 / 终止 / 重启 / 分享）
- 主体分两栏（中分线可拖动）：
  - **左栏「时间线」**：步骤瀑布流，每个步骤是一张可折叠卡片
    - 步骤类型：Thinking（思考）/ Tool Call（工具调用）/ Human Input（用户输入）/ Output（输出片段）
    - Tool Call 卡按工具种类有不同样式：File Read 显示文件路径与预览、Bash 显示命令和输出、Edit 显示 Diff（左旧右新）
  - **右栏「实时输出」**：流式 Markdown，支持代码高亮、表格、Mermaid

### 4.3 流式机制

- 接口走 SSE
- 后端按 step 推送 `step.start` / `step.delta` / `step.end` / `task.done` 事件
- 前端要求：自动重连（断网 5 秒内静默重连）、增量 Markdown 渲染（不能每次全量重渲）

### 4.4 控制能力

- 暂停：发送 `task.pause`，状态变 paused，按钮变继续
- 中途插入指令：弹出输入框，提交后作为 Human Input 步骤插入时间线
- 终止：弹二次确认后调用 `task.cancel`

### 4.5 回放

- 已完成 / 失败的任务，时间线上方出现进度滑块
- 拖动滑块可定位到任意时刻的状态快照（前端纯 UI 重放，不调后端）

### 4.6 通知

- 任务完成 / 失败时，浏览器 Notification + 站内角标
- 用户切到其他页面时仍要保持长连接（最少 30 分钟）

### 4.7 分享

- 已完成的任务点"分享"，生成只读链接，链接 7 天有效

## 5. 性能要求

- 首屏 LCP < 1.5s
- 时间线步骤数 > 200 时滚动 FPS ≥ 50
- 流式输出延迟 < 200ms（SSE 推送到屏幕渲染）

## 6. 兼容性

- Chrome / Edge / Safari 最新两版
- 最小窗口宽度 1280px

## 7. 不在本期范围

- 移动端
- 多人协作（多人观看同一个任务）
- 任务模板市场

## 8. 上线时间

期望 4 周内完成 MVP 上线。
