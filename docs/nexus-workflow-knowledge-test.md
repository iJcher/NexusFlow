# NEXUS 工作流知识库测试文档

## 文档目的

本文档用于验证 NEXUS 工作流的知识库检索、变量传递、条件分支、LLM 生成和 Reply 输出是否能够形成完整闭环。

建议将本文档上传到 NEXUS 知识库，并在工作流中使用 `KnowledgeNode` 检索本文档内容。

## 产品定位

NEXUS 是一个可视化 AI 工作流编排平台，目标是让用户像搭积木一样构建 AI 应用流程。

NEXUS 支持从空白画布开始搭建工作流，也支持从模板库选择预置模板后再修改。用户可以根据业务需要组合 Start、Knowledge、LLM、HTTP、Condition、Assign、JS Code 和 Reply 等节点。

## 核心能力

NEXUS 的核心能力包括：

1. 可视化工作流编排：用户可以在画布上添加节点、连接节点，并通过节点之间的连线定义执行顺序。
2. 知识库检索：Knowledge 节点可以根据用户输入检索已上传文档，适合构建企业知识问答、产品文档问答和客服助手。
3. 大模型生成：LLM 节点可以调用已配置的模型，根据系统提示词、用户提示词、知识库结果和上游节点输出生成回答。
4. 条件分支：Condition 节点可以根据用户输入或上游节点结果决定执行哪条路径。
5. 外部接口调用：HTTP 节点可以请求外部 API，并将响应结果传递给后续节点。
6. 变量传递：节点配置中可以使用变量选择器插入 `{{sys.query}}` 或上游节点输出，减少手写变量带来的错误。
7. 运行前校验：保存或运行前，系统会检查工作流是否缺少入口、缺少终点、存在断线、条件分支未连接或关键配置为空。

## 推荐生产级测试流程

推荐使用以下工作流测试功能流通性：

```text
Start
  -> HTTP
  -> Condition
      -> Knowledge
          -> LLM
              -> Reply
      -> LLM
          -> Reply
```

该流程可以同时验证外部接口调用、条件分支、知识库检索、大模型生成和最终回复。

## 测试问题与期望答案

### 问题一

用户问题：

```text
请介绍一下 NEXUS 的产品能力
```

期望行为：

工作流应该命中 Condition 的知识库分支，进入 Knowledge 节点检索本文档，然后由 LLM 总结 NEXUS 的核心能力。

期望回答要点：

- NEXUS 是可视化 AI 工作流编排平台。
- 支持模板创建和空白画布自由搭建。
- 支持 Knowledge、LLM、HTTP、Condition、Assign、JS Code、Reply 等节点。
- 支持变量传递和运行前校验。

### 问题二

用户问题：

```text
NEXUS 支持 HTTP 外部调用吗？
```

期望行为：

工作流可以结合 HTTP 节点执行结果和知识库内容回答。

期望回答要点：

- NEXUS 支持通过 HTTP 节点调用外部接口。
- HTTP 节点可以配置请求方法、URL、查询参数、请求头、请求体和超时时间。
- HTTP 响应可以作为后续 LLM 节点的输入。

### 问题三

用户问题：

```text
NEXUS 如何避免工作流配置错误？
```

期望行为：

工作流应该检索到运行前校验相关内容。

期望回答要点：

- 保存或运行前会进行工作流校验。
- 校验内容包括入口节点、终点节点、断线、不可达节点、非法连线和节点必填配置。
- Condition 节点的每个条件分支和 ELSE 分支都需要连接后继节点。

### 问题四

用户问题：

```text
你好，帮我写一句欢迎语
```

期望行为：

该问题不属于 NEXUS 产品能力问题，可以走 Condition 的 ELSE 分支，由通用 LLM 节点直接回答。

期望回答示例：

```text
欢迎使用 NEXUS，让我们一起高效构建智能工作流。
```

## 工作流配置建议

### Knowledge 节点

建议配置：

```text
Knowledge Base: 选择上传了本文档的知识库
Query: {{sys.query}}
Top K: 5
Threshold: 0.1
```

### Condition 节点

建议添加一个条件：

```text
变量: sys.query
操作符: contains
值: NEXUS
```

命中条件时进入 Knowledge 分支，未命中时进入 ELSE 分支。

### LLM 节点

知识库分支的 LLM 提示词建议：

```text
你是 NEXUS 产品助手。

用户问题：
{{sys.query}}

请优先依据知识库检索结果回答。
如果知识库中没有相关信息，请说明“当前知识库中没有找到明确依据”。
回答要简洁、准确、适合产品演示。
```

ELSE 分支的 LLM 提示词建议：

```text
用户问题：
{{sys.query}}

这是一个普通问题，请直接、简洁、友好地回答。
```

## 验收标准

当以下现象都成立时，可以认为工作流核心功能已经流通：

1. 保存工作流时没有结构校验错误。
2. 运行测试问题一时，执行日志包含 Knowledge、LLM 和 Reply。
3. 运行测试问题二时，执行日志包含 HTTP，并且 LLM 能引用外部调用结果。
4. 运行测试问题四时，流程进入 ELSE 分支。
5. Reply 节点能把最终答案返回到聊天测试窗口。
