# UI Design Reference Router

## 目标

把模糊的“做个好看的界面”转成可执行的设计规范，核心是：

1. 识别用户业务场景（后台管理 / 博客期刊 / 产品落地页 / 深色科技等）。
2. 识别色彩与视觉偏好（暗黑、玻璃拟态、极简白底、强排版）。
3. 在 5 套 reference 中选择 1 套主风格 + 1 套可选修饰规则。
4. 输出结构化设计需求（颜色、排版、组件、禁用项、验收检查）。

## 触发时机

当用户出现以下意图时，自动应用本 Skill：

- 想要“界面风格建议 / UI 方案 / 视觉搭配”
- 提到“后台管理系统、博客、期刊、落地页、SaaS”
- 明确提出“深色、暗黑、玻璃拟态、极简、杂志感”
- 希望“按颜色偏好给参考规范”

## 执行流程（必须按顺序）

### 步骤 1：提取需求信号

**目的**：先判断“场景”和“风格”，避免直接给错方向。

从用户话术中提取：

- **产品类型**：admin / blog / editorial / landing / dashboard
- **视觉风格**：dark / glass / minimal / editorial
- **色彩偏好**：blue / green / violet / monochrome / unknown
- **约束词**：如“不要渐变”“只要一种强调色”“企业级清晰”

若信息不完整，最多追问 2 个关键问题：

1. 这是后台管理系统、博客期刊，还是产品落地页？
2. 你偏好的主强调色是蓝、绿、紫，还是尽量无强调色？

### 步骤 2：选择主 reference

**目的**：在单一主风格下做设计决策，保证一致性。

使用以下映射：

- **暗黑产品界面** -> `暗黑UI`
- **企业后台 / 数据看板 / 管理端** -> `后台管理`
- **博客 / 长文阅读 / 内容品牌页** -> `期刊排版`
- **深色科技感 / AI / 可视化产品** -> `深色玻璃拟态`
- **SaaS 官网 / 产品营销页** -> `极简SaaS落地页`

若同时命中多个风格，优先级：

1. 产品类型强约束（admin/blog/landing）
2. 用户显式风格关键词（dark/glass/editorial/minimal）
3. 色彩偏好

### 步骤 3：做颜色匹配

**目的**：在不破坏风格规则前提下满足用户色彩偏好。

颜色决策规则：

- 每个方案只允许 **1 个 accent**
- 用户偏好与 reference 不冲突则直接采用
- 冲突时优先保留 reference 规则，并给出“可替代色”
- 禁止引入第二强调色来“折中”

### 步骤 4：输出结构化设计需求

**目的**：让设计与前端都能直接执行，不需要二次翻译。

输出固定包含 6 块：

1. **推荐方案**（主 reference + 选择理由）
2. **色彩系统**（背景/文本/边框/accent/语义色）
3. **排版系统**（字体、字号、行高、字重）
4. **组件规则**（按钮、卡片、表格、表单、导航）
5. **禁用项**（明确不要做什么）
6. **验收清单**（5-10 条可检查标准）

### 步骤 5：给页面级落地建议（可选）

**目的**：从“设计系统”落到“具体页面布局”。

根据页面类型补充结构建议：

- Dashboard：侧边栏 / 顶栏 / 卡片区 / 表格区
- Blog：Hero 标题 / 正文列宽 / Pull Quote / 分割线
- Landing：Hero / Feature Grid / Social Proof / CTA

## 参考规范来源

完整的 5 套规则放在：

- [reference.md](reference.md)

使用方式：

- 必须先选主 reference，再引用对应章节
- 不得跨章节混用冲突规则（如“极简白底”与“深色玻璃”）

```markdown

- 选择理由：<产品类型 + 风格偏好 + 色彩偏好匹配说明>

## 色彩系统
- 背景：...
- 文本：...
- 边框：...
- Accent（仅一种）：...
- 语义色（如有）：...

- 字体：...

## 组件与交互规则
- 导航：...
- 卡片/容器：...
- 表单/按钮：...
- 表格/列表：...
- Hover/Focus：...

## 禁用项
- ...
- ...

## 页面级建议（<页面类型>）
- ...
- ...

## 验收清单
- [ ] 是否只使用 1 个 accent
- [ ] 是否遵守 reference 的禁用项
- [ ] 是否满足字号、间距、对比度规则
```

## 示例映射（高频）

- 用户说“企业后台，蓝色，清晰专业”：
  - 主方案：`后台管理`
  - Accent：`#3b82f6`

- 用户说“技术博客首页，想要杂志感，克制配色”：
  - 主方案：`期刊排版`
  - Accent：`none` 或 `#1d3557`

- 用户说“AI 产品官网，要深色高级感”：
  - 主方案：`深色玻璃拟态`
  - Accent：`#8b5cf6` 或 `#818cf8`

## 禁止事项

- 不要在同一页面混用 2 套互斥 reference
- 不要给出未在 reference 中定义的渐变/阴影风格
- 不要输出“看情况”式模糊建议，必须落到具体 token
- 不要忽略用户显式约束词（如“不要渐变”“只要蓝色”）
